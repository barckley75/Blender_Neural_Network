# Blender Neural Network

Create and train artificial neural networks inside Blender, visualized as 3D geometry.

![MNIST](files/Image.GIF)

Now targeting **Blender 5.1+** with **Geometry Nodes** and **PyTorch**. The 2021 version that used Animation Nodes + TensorFlow is still in the git history; this one is a clean-sheet rewrite.

## What it does

- Renders a feed-forward neural network as 3D geometry: neurons as instanced meshes, layers spaced along the X axis, connections as curves between adjacent layers.
- Every visual parameter (neuron count per layer, grid width, mesh type, mesh size, connection radius, spacing) is live-editable from a sidebar panel and drives a Geometry Nodes modifier.
- Trains the network on a user-supplied dataset using PyTorch, running in a modal timer so the Blender UI stays responsive.

Example configurations:

- Input 3, Hidden 4, Output 2 — a simple classifier
- Input 784, Hidden 100, Output 10, Input Grid Width 28 — MNIST-scale, inputs laid out as a 28×28 grid

## Install

### 1. Install PyTorch into Blender's Python

PyTorch has to be installed into Blender's bundled Python, not the system Python. Find your Blender Python binary:

- **macOS**: `/Applications/Blender.app/Contents/Resources/5.1/python/bin/python3.11`
- **Linux**: `<blender install>/5.1/python/bin/python3.11`
- **Windows**: `<blender install>\5.1\python\bin\python.exe`

Then:

```bash
"<path to blender python>" -m pip install --upgrade pip
"<path to blender python>" -m pip install -r requirements.txt
```

This installs `torch` and `numpy`. The CPU build of PyTorch is ~200 MB.

### 2. Build the Geometry Nodes tree

Open Blender 5.1. In the Scripting workspace, open `build_gn_tree.py` from this repo and click Run Script. A file `neural_network/NeuralNetwork.blend` will be written.

Re-run this any time the spec in `docs/gn_tree_spec.md` changes.

### 3. Install the addon

Zip the `neural_network/` folder (with `NeuralNetwork.blend` inside it), then in Blender:

- Edit → Preferences → Add-ons → Install → pick the zip.
- Enable "Artificial Neural Network".

## Use

1. `View3D → Add → Neural Network` adds a new NN object with the Geometry Nodes modifier attached.
2. Open the sidebar (`N`) in the 3D viewport; there's a new `NN` tab with four panels:
   - **Neurons Aspect** — mesh type (0=Ico, 1=Sphere, 2=Cube), mesh size, connection visibility/radius, spacing.
   - **Layer Sizes** — neuron count per layer. Set hidden layers (L2, L3) to 0 to skip them.
   - **Grid Width** — how many columns to lay each layer into. Use 1 for a line; `sqrt(N)` for a square.
   - **Training (PyTorch)** — dataset path, epochs, minibatch, learning rate, Start button.

### Dataset format

Point the Training panel at a file in any of these formats:

- **`.npz`** with arrays `X` (shape `[N, features]`, float) and `y` (shape `[N]`, int labels).
- **`.csv`** with features in columns and the class label in the last column.
- **`.pt`** containing either `{"X": ..., "y": ...}` or a `(X, y)` tuple.

The number of features must equal your Input Size; the number of classes must be ≤ Output Size. The loader auto-splits 80% train / 20% validation.

## Repository layout

```
neural_network/          Addon package
  __init__.py            Registers the addon
  panel.py               Four sidebar panels
  operators.py           Bootstrap operator (Add > Neural Network)
  training_operator.py   Modal training operator
  nn_training.py         PyTorch backend (model + trainer + dataset loader)
  NeuralNetwork.blend    Bundled GN tree (generated by build_gn_tree.py)
build_gn_tree.py         Script run inside Blender to generate the .blend
tests/
  validate_tree.py       Asserts node-group interface (run in Blender --background)
  test_training.py       PyTorch backend tests (plain Python)
  test_bootstrap.py      Addon integration test (run in Blender --background)
docs/
  gn_tree_spec.md        Geometry Nodes tree specification
tasks/
  todo.md                Implementation checklist
```

## Testing

Plain Python (needs `torch` + `numpy` installed):

```bash
python -m pytest tests/test_training.py
```

Inside Blender:

```bash
blender --background neural_network/NeuralNetwork.blend --python tests/validate_tree.py
blender --background --python tests/test_bootstrap.py
```

## Status & roadmap

This is the 1.0 Blender 5.1 rewrite. Known limitations in this release:

- Connection rendering is currently a placeholder (draws an empty curve set) — layers render correctly, but the all-pairs curve generator between layers is the next thing on the list. The `Connection Visibility` / `Connection Radius` controls are wired up and will take effect once the all-pairs generator lands.
- Text-neuron aspect (old addon's option 2) is dropped; aspect 2 is now Cube.
- Live activation visualization (color neurons by forward-pass magnitudes during training) is a planned stretch goal, not shipped.

Contributions welcome — the old 2021 version had the same invitation and a few forks showed up; same applies here.
